# Requirement document
## Functional requirement
1.ระบบต้องสามารถแสดงจำนวนที่จอดรถว่างทั้งหมดในห้างสรรพสินค้าได้

2.ระบบต้องสามารถแสดงสถานะที่จอดรถแต่ละช่อง(ว่าง/ไม่ว่าง)ได้

3.ระบบต้องอนุญาตให้ผู้ใช้จองที่จอดรถล่วงหน้าได้

4.ระบบต้องอนุญาตให้ผู้ใช้ยกเลิกการจองที่จอดรถได้

5.ระบบต้องปรับสถานะที่จอดรถเป็นว่างโดยอัตโนมัติเมื่อหมดเวลาการจอง

6.ระบบต้องสามารถบันทึกข้อมูลรถที่เข้ามาจอดในลานจอดรถได้

7.ระบบต้องสามารถตรวจสอบจำนวนรถที่จอดอยู่ในลานจอดรถทั้งหมดได้

8.ระบบต้องสามารถคำนวณค่าธรรมเนียมการจอดรถของรถแต่ละคันได้

9.ระบบต้องสามารถแสดงค่าธรรมเนียมการจอดรถให้ผู้ใช้ทราบได้

10.ระบบต้องอนุญาตให้ผู้ดูแลระบบ (Admin) เพิ่มหรือจัดการจำนวนที่จอดรถในระบบได้

## non-functional requirement
1.ระบบต้องมีความถูกต้องและเชื่อถือได้ในการแสดงข้อมูลที่จอดรถ

2.ระบบต้องสามารถรองรับผู้ใช้งานหลายคนพร้อมกันได้

3.ระบบต้องมีความปลอดภัยในการจัดเก็บข้อมูลผู้ใช้และข้อมูลการจอง

4.ระบบต้องตอบสนองการทำงานได้รวดเร็วแบบ real-time หรือใกล้เคียง

5.ระบบต้องใช้งานง่ายและมีหน้าจอแสดงผลที่เข้าใจได้ง่ายสำหรับผู้ใช้ทั่วไป
