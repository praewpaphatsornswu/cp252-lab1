# Requirement document
## Functional requirement
1.ระบบต้องสามารถแสดงจำนวนที่จอดรถว่างทั้งหมดในห้างสรรพสินค้าได้

2.ระบบต้องสามารถแสดงสถานะที่จอดรถแต่ละช่อง(ว่าง/ไม่ว่าง)ได้

3.ระบบต้องอนุญาตให้ผู้ใช้จองที่จอดรถล่วงหน้าได้

4.ระบบต้องอนุญาตให้ผู้ใช้ยกเลิกการจองที่จอดรถได้

5.ระบบต้องปรับสถานะที่จอดรถเป็นว่างโดยอัตโนมัติเมื่อหมดเวลาการจอง

6.ระบบต้องสามารถบันทึกข้อมูลรถที่เข้ามาจอดในลานจอดรถได้

7.ระบบต้องสามารถตรวจสอบจำนวนรถที่จอดอยู่ในลานจอดรถทั้งหมดได้

8.ระบบต้องสามารถคำนวณค่าธรรมเนียมการจอดรถของรถแต่ละคันได้

9.ระบบต้องสามารถแสดงค่าธรรมเนียมการจอดรถให้ผู้ใช้ทราบได้

10.ระบบต้องอนุญาตให้ผู้ดูแลระบบ (Admin) เพิ่มหรือจัดการจำนวนที่จอดรถในระบบได้

## non-functional requirement
1.ระบบต้องมีความถูกต้องและเชื่อถือได้ในการแสดงข้อมูลที่จอดรถ

2.ระบบต้องสามารถรองรับผู้ใช้งานหลายคนพร้อมกันได้

3.ระบบต้องมีความปลอดภัยในการจัดเก็บข้อมูลผู้ใช้และข้อมูลการจอง

4.ระบบต้องตอบสนองการทำงานได้รวดเร็วแบบ real-time หรือใกล้เคียง

5.ระบบต้องใช้งานง่ายและมีหน้าจอแสดงผลที่เข้าใจได้ง่ายสำหรับผู้ใช้ทั่วไป

## TOR


## 1. วัตถุประสงค์ของโครงการ

เพื่อพัฒนาระบบบริหารจัดการลานจอดรถอัจฉริยะ ที่สามารถแสดงข้อมูลสถานะที่จอดรถแบบเรียลไทม์  
รองรับการจองที่จอดรถ การคำนวณค่าธรรมเนียม และอำนวยความสะดวกแก่ผู้ใช้งานและผู้ดูแลระบบอย่างมีประสิทธิภาพ

---

## 2. ขอบเขตการดำเนินงานของระบบ

ระบบแบ่งการทำงานออกเป็น 2 ส่วนหลัก ได้แก่

- ระบบสำหรับผู้ใช้งานทั่วไป (**Front-End**)  
- ระบบสำหรับผู้ดูแลระบบ (**Back-End**)

---

## 3. คุณลักษณะของระบบสำหรับผู้ใช้งาน (Front-End)

### 3.1 ระบบแสดงข้อมูลลานจอดรถ

- **3.1.1** ระบบต้องสามารถแสดงจำนวนที่จอดรถทั้งหมดภายในลานจอดได้  
- **3.1.2** ระบบต้องสามารถแสดงสถานะของที่จอดรถแต่ละช่อง  
  *(ว่าง / ไม่ว่าง)* ได้แบบเรียลไทม์

---

### 3.2 ระบบจองที่จอดรถ

- **3.2.1** ระบบต้องอนุญาตให้ผู้ใช้งานสามารถจองที่จอดรถล่วงหน้าได้  
- **3.2.2** ระบบต้องอนุญาตให้ผู้ใช้งานสามารถยกเลิกการจองที่จอดรถได้  
- **3.2.3** ระบบต้องปรับสถานะที่จอดรถเป็น **“ว่าง”** โดยอัตโนมัติเมื่อหมดเวลาการจอง

---

### 3.3 ระบบคำนวณและแสดงค่าธรรมเนียม

- **3.3.1** ระบบต้องสามารถคำนวณค่าธรรมเนียมการจอดรถของรถแต่ละคันได้  
- **3.3.2** ระบบต้องสามารถแสดงค่าธรรมเนียมการจอดรถให้ผู้ใช้งานทราบได้อย่างชัดเจน

---

## 4. คุณลักษณะของระบบสำหรับผู้ดูแลระบบ (Back-End)

### 4.1 ระบบจัดการข้อมูลลานจอดรถ

- **4.1.1** ระบบต้องสามารถบันทึกข้อมูลรถที่เข้ามาจอดในลานจอดได้  
- **4.1.2** ระบบต้องสามารถตรวจสอบจำนวนรถที่อยู่ในลานจอดทั้งหมดได้  

---

### 4.2 ระบบจัดการโดยผู้ดูแลระบบ (Admin)

- **4.2.1** ระบบต้องอนุญาตให้ผู้ดูแลระบบสามารถ  
  เพิ่ม / แก้ไข / ลบ ข้อมูลจำนวนที่จอดรถได้  
- **4.2.2** ระบบต้องอนุญาตให้ผู้ดูแลระบบสามารถกำหนดอัตราค่าธรรมเนียมการจอดรถได้  

---

## 5. ข้อกำหนดคุณลักษณะที่ไม่ใช่เชิงหน้าที่  
*(Non-Functional Requirements)*

- **5.1** ระบบต้องมีความถูกต้องและความเชื่อถือได้ในการแสดงข้อมูลสถานะที่จอดรถ  
- **5.2** ระบบต้องสามารถรองรับผู้ใช้งานหลายคนพร้อมกันได้  
- **5.3** ระบบต้องมีความปลอดภัยในการจัดเก็บข้อมูลผู้ใช้งานและข้อมูลการจอง  
- **5.4** ระบบต้องตอบสนองการทำงานได้อย่างรวดเร็ว หรือใกล้เคียงแบบเรียลไทม์  
- **5.5** ระบบต้องมีหน้าจอการใช้งานที่เข้าใจง่าย และใช้งานได้สะดวกสำหรับผู้ใช้งานทั่วไป
